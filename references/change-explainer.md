# Change Explainer Rules

Use these sources in order:

1. user request and recent conversation summary
2. git diff or changed files
3. current repository structure
4. explicit plan/todo/checklist files

Required sections:

- `지금 한 일 한눈에`
- `전에는 / 지금은`
- `사용자 입장에서 달라진 점`
- `이 프로젝트가 하는 일`
- `기능 지도`
- `용어 번역`

When the change is structural:

- explain what is easier to understand, safer to change, or more reusable now

When the change is visible:

- explain what a non-developer would notice without reading code

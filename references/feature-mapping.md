# Feature Mapping Rules

Map repository areas to user-facing capability buckets, not raw folders.

Default buckets:

- `화면과 경험`
- `처리 규칙과 데이터 흐름`
- `공통 기능`
- `문서와 이해 자료`
- `실행/관리 도구`
- `검증 장치`
- `스킬/설명 규칙`

Each bucket should answer:

- what the user feels or sees
- what kind of responsibility this area has
- which representative files best explain it

Do not lead with folder topology unless the user explicitly asks for it.

# Non-Dev Language Rules

- Lead with what changed in the product or workflow, not with file names.
- Avoid jargon unless you immediately translate it.
- Prefer:
  - `화면` instead of `UI`
  - `처리 규칙` or `운영실` instead of `server`
  - `공통 부품` instead of `shared package`
  - `검증 장치` instead of `tests`
- If a change has no user-visible effect, say that directly.
- If the work is mostly cleanup or structure, say it makes future changes safer or clearer.
- Write for someone who can use software but does not read code.
